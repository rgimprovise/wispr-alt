**Контекст:** работающий MVP под именем wispr-alt уже собран и стабилен (см. tag v0.1.2 на `main`). Сейчас на ветке `develop` начинается визуальный ребрендинг под бренд **Беловик**. Базовая ALRCV-система от прошлой итерации (тёмная, с REC-красным) **полностью заменяется** новым визуальным языком.

## 1. Продукт в одном абзаце

**Беловик** — голосовой интерфейс для работы с текстом. Российская адаптация Wispr Flow: пользователь нажимает горячую клавишу (или вызывает overlay на телефоне), говорит, и текст автоматически вставляется в активное поле — с очисткой от слов-паразитов и расстановкой пунктуации. Главное отличие от зарубежных аналогов: **оптимизация под русский язык и российские рабочие сценарии** (деловые письма, Telegram-посты, заметки, юридические тексты, ТЗ).

## 2. Платформа бренда (из брендбука)

- **Категория:** Голосовой интерфейс для работы с текстом
- **Обещание:** Из устной мысли — в готовый текст
- **Для кого:** Предприниматели, менеджеры, юристы, маркетологи, команды
- **Отстройка:** Российская адаптация под язык, привычки и рабочие сценарии РФ
- **Tone of voice:** Спокойно, точно, чисто, делово, без суеты

## 3. Целевая аудитория

**Primary:**
- Предприниматели, продакт-менеджеры, специалисты в IT/маркетинге, 25–45 лет
- Печатают много (email, Slack, Notion, Telegram, IDE)
- Уже пробовали голосовой ввод Apple/Yandex/Google, не хватает качества или гибкости стилей
- Готовы платить $5–15/мес за инструмент экономящий 30+ минут печати в день

**Secondary:**
- Юристы, врачи, журналисты — длинные диктовки, профессиональная лексика
- Создатели контента — Telegram-каналы, посты, скрипты для видео
- Корпоративные команды — единые шаблоны, командные словари

## 4. Платформы

| Платформа | Состояние | Особенности |
|-----------|-----------|-------------|
| **macOS** | MVP стабилен, нужен ребрендинг | Tauri WebView (Safari engine). Нативный feel приоритетен |
| **Windows** | MVP собирается, нужен ребрендинг | Tauri WebView2 (Chromium). Тот же CSS, без переписывания |
| **Android** | MVP overlay-driven работает, нужен ребрендинг + UX-улучшения | Native Kotlin Views (XML/programmatic). НЕ Compose в overlay (ограничения WindowManager) |
| **iOS** | Scaffold есть, в beta-listе пока нет (нужен Apple Developer Program) | SwiftUI main app + UIKit keyboard extension |

**Кросс-платформенность:** один визуальный язык, токены экспортируются в CSS (Tauri) и в XML drawables / SwiftUI tokens.

## 5. Что нужно задизайнить

### 5.1. App icon — приоритет 1

Логотип «Б» из брендбука как иконка приложения для всех платформ. Нужны варианты:
- Адаптивная Android-иконка: foreground + background (XML adaptive icon)
- macOS .icns (1024×1024 master, scaling вниз)
- Windows .ico (мульти-размер)
- iOS AppIcon set (все размеры)
- Менюбар-иконка macOS (template, 16×16 monochrome)
- Notification small icon Android (24×24 monochrome white-on-transparent)
- Quick Settings tile icon Android (24×24)

### 5.2. Overlay pill (macOS + Windows) — приоритет 1

**Сердце продукта на десктопе.** Сейчас тёмный с красной точкой. Нужна light Беловик-версия:
- Состояния: `idle` (скрыт по умолчанию), `recording` (активный), `transcribing` (обработка), `error`
- Вылетает сверху по центру экрана, не крадёт фокус
- Размер компактный, ~460×72 при простом состоянии, может расширяться при показе live-транскрипта
- Поддержка teletype-скролла текста (live partial во время записи)
- Должен работать поверх full-screen приложений (особенности реализации в коде, дизайнеру не критично)

Состояния-индикаторы:
- Idle dot: нейтральный графит/серебристый
- Recording: акцентный графит/чернильный (НЕ красный — брендбук этого избегает)
- Transcribing: мягкий навигационный/мятный

**Платформенные нюансы:**
- **macOS:** допустима стеклянная подложка (vibrancy), скруглённые углы 24–28px, тонкая полупрозрачная граница. Match с Sequoia / Liquid Glass-эстетикой
- **Windows 11:** Mica/Acrylic подложка через WebView2 (с fallback на solid если не поддерживается), скруглённые углы 16–20px (чуть жёстче чем mac), Fluent-стиль теней

Дизайнер: одна основа дизайна с минимальными вариациями для каждой ОС. Не два разных дизайна — единый язык.

### 5.3. Главное окно — macOS + Windows — приоритет 2

Сейчас минимальное dev-окно. Нужно полноценное Settings/Control window:
- Sidebar layout с секциями: **Главное / Стили / Аккаунт / История / Об отношении**
- **Главное:** выбор микрофона, hotkey picker, режим записи (toggle/PTT), дефолтный язык
- **Стили обработки:** список из 6+ режимов (Деловой / Краткий / Telegram / Письмо / Задача / ТЗ / Свободный) + редактор пользовательских стилей
- **Аккаунт:** email, тариф, остаток квоты, billing-портал ссылка, BYOK-поле
- **История:** последние транскрипции (timestamp, длительность, превью), поиск, копирование
- **Об отношении:** версия, проверка обновлений, ссылки на сайт/документацию/поддержку

Размер ~720×560, не resizable. Light тема первична, dark parity готова.

**Платформенные нюансы:**
- **macOS:** sidebar в стиле System Settings (translucent, иконки в кружках), title bar совмещён с контентом (Tauri использует `titleBarStyle: hidden`), traffic lights в углу
- **Windows:** Fluent-style sidebar (плотнее, без translucent), стандартные window controls (minimize/maximize/close) с акцентным цветом

Желательны **готовые макеты обоих ОС** — а не «один десктопный» — чтобы покрыть платформенную специфику.

### 5.4. Onboarding flow — все платформы — приоритет 2

4–5 экранов первого запуска. Один визуальный язык, но **разные шаги на разных ОС** (разные системные диалоги permissions):
1. **Приветствие** — лого, tagline, hero-визуал (3D Б или стеклянная волна из брендбука). Одинаковый на всех платформах
2. **Микрофон** — запрос runtime-permission. Универсальный экран
3. **Доступы системы** — этот экран ветвится:
    - **macOS:** «Разрешить Accessibility и Automation» со скриншотами System Settings
    - **Windows:** «Внесите wispr-alt в исключения Windows Defender» со скриншотами (если SmartScreen блочит)
    - **Android:** «Display over other apps» + «Accessibility service» + «Notifications»
    - **iOS:** «Разрешите клавиатуру» + «Allow Full Access»
4. **Первая диктовка** — guided tour, мини-демо
5. **Готово** — переход на основной экран + краткие подсказки по hotkey/жесту

Tone of voice: «Говорите свободно. Получайте чистый текст». Минимум технических терминов, дружелюбно.

Нужны макеты **всех 5 экранов × 2 темы (light/dark)** + **варианты Шага 3 для каждой платформы**.

### 5.5. Tray / menu bar (macOS + Windows) — приоритет 3

Иконка в строке меню (mac) / system tray (Windows):
- **macOS:** Б в template-стиле, монохром, адаптируется к dark/light режиму системной строки меню. Active-состояние (запись идёт) — тонкая графитовая обводка или альтернативный glyph
- **Windows:** та же идея для system tray. Поддержка сменяющихся состояний (idle/recording/transcribing) через цвет иконки или индикатор-точку

Popup-меню при клике/правом клике (одинаковое на обеих платформах, стилизация — нативная):
- Текущий статус (записывается / готов)
- Quick action: Начать запись (показывает текущий hotkey)
- Последние 5 транскрипций (копировать в один тап)
- Открыть Историю / Настройки
- Выход

Нужны макеты popup-меню для **обеих платформ** (стилизация чуть разная — macOS round corners, Windows squarer).

### 5.6. Android — приоритет 1 (одновременно с overlay)

**MainActivity** (главное окно приложения):
- Onboarding 4 шага (микрофон, overlay-permission, accessibility, готово)
- Состояние: enabled/disabled toggle с визуальным индикатором
- Подсказки как использовать (QS tile / уведомление / floating bubble)

**Overlay pill** (поверх любого приложения) — light Беловик-версия, та же логика что desktop overlay но адаптированная под mobile (большие тач-области, чтение одной рукой)

**Floating bubble** — постоянный плавающий пузырёк (как Messenger heads), тап = разворачивает в overlay pill. Альтернатива QS tile для пользователей которые предпочитают всегда-видимый триггер

**Notification** в шторке — постоянное low-priority уведомление, action button «Начать диктовку»

### 5.7. iOS keyboard extension — приоритет 4 (отложить)

Текущий scaffold переразработать под брендбук, но это после macOS+Android+Windows.

### 5.8. Empty states / errors

- Нет интернета
- Backend недоступен
- Слишком короткая запись («слишком тихо или слишком коротко»)
- Превышена квота
- Ошибка авторизации
- Микрофон занят другим приложением

## 6. Визуальный язык — из брендбука

(Не переизобретать — следовать PDF и расширять там где нужны конкретные значения)

**Палитра:**
- Чистый белый — основной фон
- Чернильный — основной текст
- Графит — акцент, активные состояния
- Мягкий серый — secondary
- Текстовый серый — tertiary (метки, подсказки)
- Мятная белизна — мягкий нейтральный фон секций
- Серебристый — границы, разделители

**Радиусы:** 24–40px на крупных контейнерах (как сказано в брендбуке про UI-принципы)

**Типографика:** заголовки — фирменный с округлёнными формами; тело — чистый sans, читаемый на 13–15px

**Графический язык** для иллюстраций:
- 3D-текстура буквы Б (heroes, app icons)
- Голосовая волна — динамичная, показывает движение
- Линии текста — статичный паттерн
- Стеклянные узоры — для overlays и фоновых акцентов

**Motion:**
- Спокойно, плавно
- 200–300ms на стандартные переходы
- Без резких snap-эффектов
- Easing: ease-out cubic-bezier(0.25, 0.46, 0.45, 0.94)

## 8. Технические ограничения, которые дизайнер должен знать

**Tauri WebView (desktop):**
- Поддерживает backdrop-filter, transforms, modern CSS, CSS custom properties
- НЕ поддерживает: некоторые нативные элементы macOS (vibrancy через CSS только, не NSVisualEffectView)
- Шрифты — system fallback (`-apple-system`, `Segoe UI`) или загруженные через `@font-face`

**Android Views:**
- Overlay pill — programmatic Kotlin (LinearLayout + GradientDrawable). XML drawables для каждого state
- MainActivity — XML layout + Material Components
- Нет Jetpack Compose в overlay (несовместимо с TYPE_APPLICATION_OVERLAY)
- Шрифты — через `res/font/` (TTF/OTF)

**Адаптация:**
- Цвета как CSS variables → экспорт в `tokens.css`
- Те же цвета как Android `colors.xml`
- Радиусы и spacing — single source of truth

## 9. Что мы хотим получить (deliverables)

1. **Tokens** — JSON или CSS variables: colors (light + dark), spacing, radii, motion durations, type scale. Готовый к копированию в код
2. **Логотип** — SVG в нескольких вариантах (полный, иконка, монохром, на тёмном фоне)
3. **Иконки приложения** — все нужные размеры/форматы по списку 5.1
4. **High-fi макеты** в Figma / Pencil:
   - **Overlay pill — macOS:** 4 состояния × 2 темы (light/dark)
   - **Overlay pill — Windows:** 4 состояния × 2 темы
   - **Главное окно — macOS:** 5 секций (Главное/Стили/Аккаунт/История/Об отношении) × 2 темы
   - **Главное окно — Windows:** 5 секций × 2 темы
   - **Onboarding (общий):** экраны 1, 2, 4, 5 — × 2 темы
   - **Onboarding (специфика permissions):** экран 3 в вариантах для macOS, Windows, Android, iOS
   - **Tray / menu bar popup — macOS:** раскрытое меню
   - **Tray popup — Windows:** раскрытое меню
   - **Android:** MainActivity (onboarding + control panel), overlay pill (4 состояния), floating bubble (свёрнутый/развёрнутый), notification stripe в шторке
   - **iOS (когда дойдём):** main app screen, keyboard extension
5. **Компонентная библиотека** — Button (primary/secondary/ghost), Input, Toggle, Tabs, Modal, ListItem, Badge, Pill, Empty state. Состояния: default/hover/active/focus/disabled
6. **Шрифты** — конкретные семейства + веса + лицензия (если коммерческие — где взять)
7. **Motion guidelines** — какие easing/duration для каких переходов. Спецификация по 1-2 ключевым анимациям (overlay enter/exit, pill scroll)
8. **Accessibility notes** — контрасты WCAG AA подтверждены, focus rings определены, prefers-reduced-motion поведение
